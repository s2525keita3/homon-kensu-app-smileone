## 1. 目的・背景

担当別訪問件数の **所定フォーマット PDF** を読み取り、職種に応じたルールで **分数・時間（件数）** を集計する。  
併せて **介護保険相当の概算売上**（支援/介護区分に応じた単価、医療件数の暫定加算）を算出する。  
汎用として **PDF 行単位 / CSV / Excel** からの明細取り込みとカテゴリ集計も行う。

---

## 2. システム概要

| モード | 概要 |
|--------|------|
| **担当別訪問件数 PDF 集計** | サイドバートグル ON で、選択ファイルの **先頭 `.pdf`** をレポートとして解析し、`report_df` を生成。 |
| **汎用取り込み** | トグル OFF、または PDF がレポート用途でない場合は行ベース明細に蓄積。 |

---

## 3. 利用環境・技術

- **実行:** Python 3、`streamlit`（`python -m streamlit run app.py`）
- **依存:** `requirements.txt`（`streamlit`, `pandas`, `pdfplumber`, `pymupdf`, `altair`, `openpyxl` 等）
- **PDF テキスト抽出:** pdfplumber 優先 → 空なら PyMuPDF → pypdfium2 の順（`report_parser`）

## 4. 機能要件

### 4.1 担当別訪問件数 PDF（集計）

- **対象ファイル:** アップロードのうち **最初の `.pdf`** のみ（複数 PDF 同時集計はスコープ外）。
- **担当者名:** 「担当者名」ヘッダー以降のブロックから、データ行左側の **姓・名** を抽出。文字化け時は代替パターンで救済。
- **件数の取得元:** 各ブロック内の **サマリー行**（例: `訪2: ○回`, `P60: ○回`）。**明細行の単純カウントは用いない**。
- **支援と介護の分割（売上・内訳用）:**  
  - 行が **`介護` のみ** の行で上下を分割：**上＝支援**（予防・介護予防を含む）、**下＝介護**。  
  - **`介護` 見出しがなく**、かつ **`支援` / `介護予防` / `予防`** のいずれかの行がある → **ブロック全体を支援**。  
  - 上記以外 → **従来どおりすべて介護側**に計上。  
  - 分割結果は内部列 `_vis2_s`〜`_p60_c` および `_pricing_split` に保持。
- **合計件数:** 支援・介護の合算。列「60」は **訪問看護3（vis3）** と **療法 P60** で共用のため、**P20=P40=0 かつ vis3==P60 かつ訪問系あり** のとき **P60 重複とみなし P60 を 0**（内部 `_p60_s/c` も 0）。
- **職種:** 訪問看護系のみ → 看護師。PT 系のみ → 療法士。両方 → **看護師・療法士**。いずれもなければ **—**（医療のみ等）。
- **医療回数:** ブロック内サマリーから複数パターンで抽出（`_extract_medical_count_from_block`）。
- **同行:** ブロック内「2回目訪問」等と、全文「副)2回目訪問」行の **max**（担当者名正規化）。
- **表示フィルタ:** サイドバーで **除外名・含める担当のみ**。集計ロジックは変えず **表示・CSV・ダッシュボード** に反映。

### 4.2 概算売上（介護保険相当 ＋ 医療暫定）

- **介護側:** `service_fees` の定数により、**支援単価**と**介護単価**を区分ごとに適用。  
  - PDF から `_vis2_s`/`_vis2_c` 等がある行は **件数ごとに加算**。  
  - 無い行（CSV 等）は `SUPPORT_RATIO_FOR_FEES` でブレンド（現状 0）。
- **医療暫定:** 列「医療」の件数 × **固定単価（円/回）** を **概算売上(円) に加算**（`add_revenue_columns`）。  
- **列「医療保険暫定(円)」:** 内訳表示用（医療件数×同単価）。CSV では「医療」の直後に配置。
- **売上/時間:** `概算売上(円) ÷ (分数合計/60)`。

### 4.3 汎用取り込み（明細）

- **PDF:** 行単位で `VisitRow`（日付・カテゴリ推定・raw）。担当者は空のまま扱う実装。
- **CSV / Excel:** 列マッピングで `VisitRow` に変換。
- **全クリア:** 明細と `report_df` を削除。

### 4.4 UI（Ver.1）

- **レイアウト:** **全幅**。上段に「担当別 集計結果」表と CSV ダウンロード、**区切りの下**に「ダッシュボード」。
- **ダッシュボード・サマリー:** **時間合計（h）**、概算売上（介護＋医療暫定）、医療暫定（内訳）、同行、平均売上/時間。**分数合計のメトリクスは表示しない**。
- **職種別・コード別:** 表・棒・円（Altair 利用可）。
- **トグル:** 「担当別訪問件数 PDF として集計する」「分数合計と計算式も表示する」。
- **計算式 OFF 時:** 内部列（`_vis3`, `_p60`, `_vis*_s/c`, `_pricing_split` 等）を非表示。

## 5. 出力仕様（担当別集計表）

| 列名 | 意味 |
|------|------|
| 担当者 | 抽出した担当者名 |
| 20 / 30 / 40 / 60 / 90 | 職種に応じた **回数**（下表） |
| 医療 | 医療の回数 |
| 医療保険暫定(円) | 医療件数×暫定単価（内訳） |
| 同行 | 2 回目訪問（同行）の回数 |
| 件数 | 分数合計÷60 を **小数第 1 位**（時間 h） |
| 概算売上(円) | 介護相当＋医療暫定 |
| 売上/時間(円/h) | 上記÷時間 |
| （内部）_職種 | 療法士 / 看護師 / 看護師・療法士 / — |
| （内部）_分数合計 | 集計用分数（分） |
| （内部）_vis3, _p60 | 訪3 と P60 の分離 |
| （内部）_vis*_s, _p*_s / _c | 支援・介護別件数 |
| （オプション）計算式 | 一次式の文字列 |

**列と職種の対応（表の数値は回数）** — 要件定義書従来どおり（20=PT20、30=訪2、…）。

---

## 6. 計算ルール（分数・件数）

- 療法士・看護師・複合で **分数式** は実装（`report_parser.summarize_report_pdf`）に準拠。  
- P40 は **36 分/回**（0.6h 相当）。  
- 同行は **1 回あたり 60 分減算**。  
- 件数 `= round(分数合計/60, 1)`。

---

## 7. 非機能要件

- PDF 文字化けに対し複数抽出エンジン・代替正規表現を用意。
- 同一担当の複数ブロックは **担当者キーで加算**。

---

## 8. スコープ外（Ver.1）

- 複数 PDF の一括集計・月の自動判別  
- スキャン PDF の OCR  
- 医療保険の明細パース結果をダッシュボードに常時表示（Ver.1 では未使用。`medical_insurance_calc` はライブラリとして残存）

## 9. 用語

| 担当別訪問件数 PDF | 「担当者名」ヘッダーとサマリー（訪2〜4、P20〜60）を含む所定レポート |
